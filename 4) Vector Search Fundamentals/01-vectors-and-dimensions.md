## What Is a Vector?

A vector is a numerical representation of unstructured data — text, images, audio. It is stored as an array of floating-point values where each value represents one **dimension**.

```
[0.023, -0.418, 0.891, 0.002, -0.334, ...]
```

The terms *vector*, *vector embedding*, and *embedding* are interchangeable and all refer to this array.

Vectors give you the ability to search and compute unstructured data as easily as structured data, because distance in vector space encodes semantic similarity.

---

## Embedding Models

Vectors are generated by **embedding models** — deep learning models trained on large corpora. You send unstructured text (or other data) to the model; it returns a vector encoding the semantic content.

### Model Categories

Not all LLMs generate embeddings. There are distinct model types:

- Text generation models
- Classification models
- **Embedding models** ← what we use here

### Voyage AI Models (used in this course)

| Model | Best For |
|---|---|
| `voyage-3.5` / `voyage-3.5-lite` | General-purpose retrieval |
| `voyage-law-2` | Legal document retrieval |
| `voyage-code-2` | Code-related tasks |
| `voyage-context-3` | RAG systems — captures chunk-level detail AND global document context in one pass |

## How Vectors Represent Meaning

Each dimension in a vector corresponds to some feature or characteristic of the data. The value in that dimension represents the strength or presence of that characteristic. You don't choose what the dimensions mean — the model learns them during training.

In a simplified 2D example, you might plot vectors for *car*, *truck*, *police car*, and *ambulance* on a graph. Similar items cluster together; distance between points reflects dissimilarity.

In practice, useful embeddings have **hundreds to thousands of dimensions** — not 2. The full vector space is not directly visualizable, but the same clustering behavior holds.

---

## Two Factors That Control Results

### 1. The Embedding Model

Different models are trained differently and place items in different positions in vector space. One model might consider a car and truck more similar than a car and police car; another model might disagree. The model's training determines the geometry.

**There is no universal best model.** You must experiment for your use case.

### 2. The Similarity Function

After embeddings are placed in space, you choose a function to measure similarity between two vectors. Options include:

- **Euclidean distance** — straight-line distance between points
- **Cosine similarity** — angle between vectors (ignores magnitude)
- **Dot product** — angle + magnitude combined

Different functions can return meaningfully different rankings for the same query. Details covered in Module 3.

---

## Key Takeaways

- Vectors are floating-point arrays; each element is one dimension.
- Embedding models generate the vectors from raw data.
- Similar items cluster together in high-dimensional vector space.
- Two independent choices control your results: the model and the similarity function.
- More dimensions does not automatically mean better results — always benchmark.
- Atlas Vector Search supports up to **4096 dimensions** (verify in current docs as this may increase).
